This is The Number Game, a project I did on my own time to practice honing in my JavaScript skills. The game itself is pretty simple, and, because I enjoy trying to make things enjoyable for the user, I added some extra features that enhance the game nicely.

About This Project

What is The Number Game?

The Number Game is a number guessing game in which the player must guess the randomly-generated winning number in the least amount of guesses possible. After each guess, the game lets the player know whether their guess was too high, too low, or just right.

Original Source

I originally got the idea for this project from a couple of different sources, but specifically the one listed above under 'Original source'. The original instructions are as follows:

Write a guessing game where the user has to guess a secret number. After every guess the program tells the user whether their number was too large or too small. At the end the number of tries needed should be printed. It counts only as one try if they input the same number multiple times consecutively.

Project Planning/Developer Notes

The project is very simple once you strip down all of the DOM manipulations. Really, all it is is:

  • Generate a random number between x and y (1 - 10, 1 - 100, etc.)
  • Get user's input somehow
  • Compare user's input with random number
    • If too high or too low, say so and add 1 to their 'Guesses' score
    • If it's the winning number, end the game and announce their score
  • Find a way to address any potential errors such as the user trying to enter invalid characters (letters, special chars, etc.), a number that's outside of the range, etc.

Because it is so simple, I added a few bells and whistles to make the game a tad more interesting and/or playable. For example, I have different color themes depending on the game's state - light blue for 'too low', dark blue for 'too high', green for 'win', and red for 'error' - and, after playing through my game a couple of times, I implemented some keyboard shortcuts so that the user didn't have to switch between their mouse and their keyboard if they didn't want to. I'm all about trying to make the user experience a good one, and if I get annoyed at something, I generally try to improve upon my code so that it's fixed.

Updated Code (jQuery)

Just to recap, in the previous section, we went over the original code for this program/game, which was written in vanilla JavaScript and was longer than it had to be due to my lack of realization that I should have been combining some of the classList code lines.

A while after initially finishing this program, I was reviewing it for my Project Breakdown post when I had the idea to refactor the code into jQuery. Not only would it reduce the number of lines that the program takes up, but it would also give me an opportunity to practice jQuery, which was still extremely new to me at the time.

When all was said and done, this was my final program:
